# 視覺化主題文案風格（Text Style）

> 適用於 `src/content/explore/*.md` 的 Markdown 正文。  
> 定義**怎麼寫**，不收錄各篇完整內文。互動工程見 [`p5toreact.md`](p5toreact.md)；canvas 視覺見 [`art.md`](art.md)。

---

## 1. 目的與邊界

| 是 | 不是 |
|----|------|
| 簡短、可掃讀的數學說明 | 完整教科書章節 |
| 引導讀者帶著問題看互動 | 逐步操作手冊（除非互動已上線） |
| 與全站一致的繁體用字 | 簡體維基條目名、中英混雜標題 |

作品集 `src/content/works/*.md` 可參考同一套章節邏輯，但允許更長的「參數方程／實作要點」；本文件以 **explore 主題頁** 為準。

---

## 2. 篇章結構（固定順序）

每篇 explore 正文依序包含以下五節，**不增減、不改名**：

```
## 基本概念
## 互動說明
## 觀察重點
## 相關作品
## 延伸閱讀
```

frontmatter 另備 `title`、`description`、`category`、`date` 等欄位（見 [`README.md`](README.md)「新增視覺化」）。

---

## 3. Frontmatter

### `description`

- **一句話**，約 25～45 字。
- 含：核心數學對象 + 讀者在互動裡要看什麼。
- 避免「待實作」、避免空泛形容（「精彩的」「有趣的」）。

```yaml
# 可
description: 以離心率 e 統一定義圓錐曲線；拖動 e 觀察圓、橢圓、拋物線與雙曲線的連續過渡。

# 避免
description: 關於二次曲線的一篇視覺化。
```

### 其他

- `category`：`幾何`｜`代數`｜`統計`｜`拓樸`｜`分析`（與 `content.config.ts` 一致）。
- `coverImage`：僅在已有封面圖時設定；路徑放 `public/explore/`。

## 4. 各節寫法

### 4.1 基本概念

1. **主公式**放在獨立 `$$ … $$` 區塊（必要時可兩個公式，前加短標籤如「等差數列：」）。
2. 公式後接 **1～2 句**繁體說明：這個式子在說什麼、符號代表什麼。
3. 不另開「概述」子標題；不把互動操作寫在這一節。

### 4.2 互動說明

- 以 **2～4 條**條列為主，格式：`- **模式或控件名稱**：一句說明`。
- 模式名稱用粗體即可；**不要**在「觀察重點」用粗體當小標題（見下）。
- 可選最後一行：**建議順序**：A → B → C（用箭頭串閱讀／操作順序）。
- 互動尚未實作時，仍用**現在式**描述預期行為（與已上線篇一致），勿寫「將來可以…」。

### 4.3 觀察重點

- **固定 3 條** `- …` 條列。
- **每條一句話**；可含符號與簡短公式。
- **禁止** `- **短標題**：說明` 這類格式；觀察重點不加粗體小標。

```markdown
# 可
- 相位差決定該點是相長干涉還是相消干涉

# 避免
- **相位與干涉**：相位差決定……
```

### 4.4 相關作品

- 一律 `- [作品標題](/works/{slug})` 條列。
- 不用「標題 —— 補充說明」的單行段落體。
- 只連結站內已有作品；順序建議：與主題最密相關者在前。

### 4.5 延伸閱讀

- 通常 **2 條**外連。
- 格式：`- [條目名（維基百科）](https://zh.wikipedia.org/zh-tw/…)`。
- 連結文字與條目名用**繁體**；優先 `zh-tw` 路徑。

## 5. 數學與用字

| 項目 | 規範 |
|------|------|
| 行內數學 | `$…$` |
| 獨立公式 | `$$…$$`，前後空一行 |
| 專有名詞 | 與站內他處一致（例：尤拉、邏輯斯諦、常態分佈、謝爾賓斯基） |
| 標點 | 中文全形；中英、中文與公式之間不需額外空格 |
| 術語強調 | 正文偶可用 `**粗體**` 標一個概念；觀察重點條列內避免粗體開頭 |

## 6. 最小範本（僅結構）

```markdown
---
title: 主題名稱
description: （一句話，見 §3）
category: 分析
date: 2026-01-01
featured: false
draft: false
---

## 基本概念

$$
\text{主公式}
$$

（一兩句說明。）

## 互動說明

- **某模式**：讀者可調整什麼、會看到什麼
- **另一模式**：……

建議順序：…… → …… → ……

## 觀察重點

- （現象或規律，一句）
- （現象或規律，一句）
- （現象或規律，一句）

## 相關作品

- [作品 A](/works/slug-a)
- [作品 B](/works/slug-b)

## 延伸閱讀

- [條目（維基百科）](https://zh.wikipedia.org/zh-tw/…)
- [條目（維基百科）](https://zh.wikipedia.org/zh-tw/…)
```
