## 1. 目的與邊界

| 是 | 不是 |
|----|------|
| 簡短、可掃讀的數學說明 | 完整教科書章節 |
| 引導讀者帶著問題看互動 | 逐步操作手冊（除非互動已上線且必要） |
| 與全站一致的繁體用字 | 簡體維基條目名、隨意中英混雜 |
| 公式 + 一句話直覺 | 空泛形容（「精彩的」「有趣的」） |

**語氣**：現在式、陳述句；互動未上線時仍寫「拖動滑桿…」，勿寫「將來可以…」「待實作」。

## 2. 兩套範本（explore / works）

兩者共用 §3～§6 的 frontmatter、數學、用字、分層規則；**章節名稱與順序不同**。

### 2.1 Explore（`/explore/[slug]`）

固定 **五節**，不增減、不改名：

```text
## 基本概念
## 互動說明
## 觀察重點
## 相關作品
## 延伸閱讀
```

### 2.2 Works（`/works/[slug]`）

固定 **四節**（互動已上線者可在「互動說明」與「相關作品」之間加「觀察重點」）：

```text
## 參數方程
## 互動說明          ← 現有「實作要點」改此名
## 觀察重點          ← 可選；已上線互動建議有，固定 3 條
## 相關作品          ← 現有「相關連結」改此名
## 延伸閱讀
```

**禁止**再使用 `## 概述`（內容併入 `## 參數方程` 首段 1～2 句）或 `## 相關連結`。

## 3. Frontmatter

### 3.1 共用

| 欄位 | 規範 |
|------|------|
| `title` | 繁體；與 H1 一致；作品名可含副題（例：尤拉公式旋轉動畫） |
| `description` | **一句話**，約 25～45 字；對象 + 互動／視覺重點 |
| `date` | 真實發布日；不得用來偽造排序 |
| `order` | 列表、首頁最新、上下篇排序唯一依據；draft 可為 `0`，發布內容必須為同 collection 內不重複的正整數 |
| `featured` | `true` = 首頁「精選」池（需互動已掛載；不影響 `/works`、`/explore` 列表排序） |
| `draft` | `false` 才發布 |

`description` 用**純文字**寫公式（例：`z_{n+1}=z_n^2+c`），不用 `$…$`。

```yaml
# 可
description: 以離心率 e 統一定義圓錐曲線；拖動 e 觀察圓、橢圓、拋物線與雙曲線的連續過渡。

# 避免
description: 關於二次曲線的一篇視覺化。
description: 待實作的朱利亞集展示。
```

### 3.2 Explore 專用

```yaml
category: 幾何   # 幾何 | 代數 | 統計 | 拓樸 | 分析
coverImage: /images/explore-covers/{slug}.png   # 可選；新封面規格見 exploreart.md
```

### 3.3 Works 專用

```yaml
tags:
  - 幾何
  - 複數
```

`tags` 自由多選；**不用** `category`。

## 4. 用字：一律繁中

**總則**：content 正文（含標題、各節、粗體小標）以**台灣繁體常用譯名**書寫。  
保留不譯的僅限：**公式符號**（$z$、$\theta$、$e$）、**已約定變數名**（滑桿旁的 $k$、$N$）、**URL**。

| 類別 | 規範 |
|------|------|
| 人名／定理 | 尤拉、棣美弗、吉布斯、傅立葉、黎曼、貝氏（不用歐拉、貝叶斯） |
| 數學物件 | 朱利亞集、曼德博集、康托爾集、邏輯斯諦映射、常態分佈 |
| 幾何／物理 | 駐波、干涉條紋、波節、波腹、極座標、複平面、行（matrix 的「行」） |
| 碎形／動力 | 分岔、自相似、迭代函數系統（不用 IFS 作正文主詞） |
| 視覺用語 | 圓周疊加（不用 epicycles）、漸顯生長（不用 reveal）、光暈（不用 glow 作動詞） |

**禁止寫進 content 的工程詞**（改寫成讀者語言，或移入 `p5toreact.md`）：

`lerp`、`tile`、`ImageData`、`GC`、`out 參數`、`cacheStrategy`、`noLoop`、`deltaTime`、Progressive Rendering、smooth iteration 等。

**維基外連**：錨文字可跟**該站條目標題**（常為「歐拉公式（維基百科）」），與正文「尤拉」並存不衝突；URL 一律 `zh-tw`。

**英文括註**（可選、僅首次）：`駐波（standing wave）`；後文只寫「駐波」。  
`description`、觀察重點、相關作品**不加**英文括註。

### 4.1 標點

- 中文全形標點。
- 中文與 `$…$` 之間不額外加空格。
- 分號 `；` 可連接 description 中前後兩個子句。

## 5. 內容分層：互動說明究竟寫什麼

你在 `## 實作要點`（以及 p5 原型文末的「實作要點」）裡，其實混了 **四種不同文本**。  
rename 成 `## 互動說明` 之後，**只保留第 1 層；第 2 層視情況；第 3、4 層移出 content**。

### 5.1 四層對照

| 層 | 你在寫什麼 | 典型句型 | 應放在 |
|----|------------|----------|--------|
| **① 控件 → 畫面** | 讀者拖哪個滑桿、切哪個模式、畫布上會變什麼 | 「調整 **Re(c)**，分形邊界即時重算」 | **`## 互動說明`** |
| **② 幾何呈現** | 用哪些輔助線／顏色幫讀者「看懂」式子（仍從讀者視角） | 「加法時顯示 $z_1$、$z_2$ 張成的平行四邊形」 | **互動說明**（1～2 條即可） |
| **③ 工程實作** | 怎麼算、怎麼畫、怎麼優化 | 「每幀 `lerp` 0.08」「tile 漸進渲染」「零配置 Temp 池」 | **`p5toreact.md`、runtime 程式** |
| **④ 數學／規律** | 定義、判據、定理後果 | 「若 $\|z_n\|>2$ 則發散」「相位差決定相消」 | **參數方程** 或 **觀察重點** |

**一句話**：互動說明 = **「我（讀者）操作什麼 → 看見什麼」**；觀察重點 = **「即使不操作也成立的數學規律與讀圖結論」**。互動說明不是 **「程式怎麼寫」**，也不是 **「定理在說什麼」**。

### 5.2 你現有文案的歸類（範例）

以下摘自站內 works；說明你原先混寫的位置。

**① 應留在互動說明（標準）**

| 原文 | 理由 |
|------|------|
| 傅立葉 explore：「**項數 N**：以滑桿調整疊加項數…」 | 控件 + 視覺結果 |
| 朱利亞集：「**參數 $c$**：滑桿調整 $c$ 的實部、虛部，即時重算分形」 | 同上 |
| 複數四則：「**雙複數滑桿**：分別調整 $r_1,\theta_1,r_2,\theta_2$」 | 同上（與現行 canvas 一致） |

**② 幾何呈現 — 可留 1～2 條在互動說明**

| 原文 | 理由 |
|------|------|
| 「**平行四邊形法**：加法時顯示…平行四邊形」 | 讀者看見的構造 |
| 「**色帶映射**：逃逸速度映射為連續色帶」 | 若改寫成「邊界依逃逸快慢呈色帶」即可 |

**③ 工程實作 — 應移出 content**

| 原文 | 應移去 |
|------|--------|
| 駐波：「**直接描點**…不做極座標轉換」 | 工程 |
| 駐波：「以 lerp（係數 0.08）平滑過渡」 | 工程 |
| 線性變換網格：「**矩陣直接映射**…不依賴外部向量函式庫」 | 工程 |
| 線性變換網格：「**網格生長**…reveal」 | 工程（正文改：「網格由中心**漸顯**展開」） |
| 原型文末：「零配置架構」「Progressive Rendering」「離屏渲染」 | `p5toreact.md` / 架構 md |
| 玫瑰曲線：「讓 $\theta$ 從 0 到 $2\pi$ 逐漸增加來描點」 | 採樣流程，非讀者操作 |

**④ 放錯章節 — 應搬家**

| 原文 | 現位置 | 應改到 |
|------|--------|--------|
| 「逃逸判斷：若 $\|z_n\|>2$ 則視為發散」 | 朱利亞集 · 參數方程 下 | 參數方程**首段**內文，不要「逃逸判斷：」小標 |
| 傅立葉：「**吉布斯現象**：…約 9% 跳躍…」 | explore · 互動說明 | **觀察重點**（這是規律，不是控件） |
| 複數四則：「**運算模式切換**：加減乘除…」 | 實作要點 | 若 canvas **尚未有**除法模式 → 刪或改為實際有的加／乘 |
| 朱利亞集：「**視圖縮放**：平移與縮放…」 | 實作要點 | 若 **尚未實作** pan/zoom → 刪；已實作才寫入 ① |

### 5.3 互動說明 vs 觀察重點（快速判斷）

```
提到滑桿、模式、拖動、切換、畫面上出現某圖層  →  互動說明
不操作也成立的數學規律、極限、對照、直覺     →  觀察重點
提到演算法、幀率、快取、lerp、模組、渲染 pass  →  工程文件
```

若同一句同時包含操作與規律，拆成兩句：操作造成的畫面變化留在「互動說明」，規律本身移到「觀察重點」。例如「拖動 k 看到奇數 k 得 k 瓣、偶數 k 得 2k 瓣」可在互動說明保留操作句，並在觀察重點寫成「花瓣數與 k 的奇偶性相關」。

### 5.4 互動說明條目格式（統一）

```markdown
- **控件或模式名（繁中）**：讀者做 X → 看見 Y
```

- **2～4 條**（works 已上線最多 5 條）。
- 粗體標籤若指涉具名控制項，**必須用讀者在畫面上看得到的字**（按鈕文字，不是程式內部
  mode 名稱或側欄區塊標題），且該控制項必須真的存在。由 `audit:explore-controls`／
  `audit:work-controls` 強制，契約見 `content-interaction-contract.md`。
- 粗體只標**控件／模式名**，不標工程名詞。
- 可選末行：**建議順序**：A → B → C（explore 必備；works 可選）。

**改寫範例（駐波 · 實作要點 → 互動說明）**

```markdown
# 混寫（舊）
- **時間推進**：t 隨幀數持續遞增，驅動波形上下振盪
- **振幅過渡**：調整 A 時以 lerp（係數 0.08）平滑過渡

# 拆分後
## 互動說明
- **振幅 A**：調大時波峰升高，包絡隨之張開
- **空間頻率 k**：改變波節與波腹數量；切換時波形由中心重新漸顯
- **時間速度 ω**：控制駐波振盪快慢

## 觀察重點
- 節點處位移恆為零，腹點振幅最大
- 包絡線 $\pm 2A\sin(kx)$ 不隨時間移動
- 兩列反向行進波疊加才會形成駐波，不是單一 traveling wave
```

（`lerp`、幀驅動等移入 `p5toreact.md` 該作品小節。）

## 6. 數學格式

| 項目 | 規範 |
|------|------|
| 行內 | `$…$` |
| 獨立公式 | `$$…$$`，前後各空一行 |
| 多公式 | 同一節內連續 `$$` 區塊可 1～2 個；前有短標籤時用正文引導（例：「設 $z_1=r_1 e^{i\theta_1}$：」） |
| 符號表 | 可在公式後用 `- **A**：說明` 條列（works 參數方程節常見） |

## 7. Explore 各節寫法

### 7.1 基本概念

1. 主公式放在 `$$ … $$`。
2. 公式後 **1～2 句**：式子在說什麼、符號意義。
3. 不寫操作步驟；不另開「概述」。

### 7.2 互動說明

- 只寫 §5 的 **① 控件→畫面** 與少量 **② 幾何呈現**。
- **2～4 條** `- **模式或控件**：一句`；格式見 §5.4。
- 末行 **建議順序**：A → B → C。
- **現在式**（佔位篇亦然）；勿寫工程詞（§4）。
- 數學規律（吉布斯、收斂性）→ **觀察重點**，勿放此節。

### 7.3 觀察重點

- **固定 3 條** `- …`。
- **每條一句**；可含符號。
- **禁止** `- **小標**：…`（不加粗體開頭）。

### 7.4 相關作品

```markdown
- [作品標題](/works/{slug})
```

- 每條一個連結；**禁止** `- 相關作品：[A](/…)、[B](/…)`。
- 可連 explore（例：`- [三角函數的疊加與波的干涉](/explore/trig-wave-interference)`），但 explore 篇優先連 works。
- 2～4 條；最相關者在前。

### 7.5 延伸閱讀

- **2 條**外連（必要時 3 條）。
- `- [條目名（維基百科）](https://zh.wikipedia.org/zh-tw/…)`。

## 8. Works 各節寫法

### 8.1 參數方程

1. **首段 1～2 句**定義物件（吸收原 `## 概述`）。
2. 主公式 `$$ … $$`（可 2 個區塊）。
3. 可選：符號說明條列 `- **k**：波數…`。
4. 不加「逃逸判斷：」這類工程段落標題；判據寫進互動說明或觀察重點。

### 8.2 互動說明（原「實作要點」）

- 與 explore 相同，依 **§5** 分層；**禁止**第 ③ 層工程句。
- **2～5 條**；只寫**已實作**的控件與畫面（勿寫規劃中模式）。
- 原「實作要點」裡的 lerp、reveal、tile、描點流程 → `p5toreact.md`。

### 8.3 觀察重點（建議，已上線互動）

- 與 explore 相同：**3 條**、一句、無粗體小標。
- 佔位作品（無 canvas）可整節省略。

### 8.4 相關作品

- 標題用 `## 相關作品`（不用「相關連結」）。
- 格式同 §7.4；可另加 explore 連結：

```markdown
## 相關作品

- [三角函數的疊加與波的干涉](/explore/trig-wave-interference)
- [干涉條紋](/works/interference-fringes)
- [克拉尼圖形](/works/chladni-figures)
```

### 8.5 延伸閱讀

- 同 §7.5；通常 1～2 條即可。
